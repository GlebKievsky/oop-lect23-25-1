1
Перевизначення методу 🔄

Уяви, що ти для гри написав клас «Корова». У ньому є багато полів і методів. Об'єкти цього класу можуть робити різні речі: йти, їсти, спати. Ще корови дзвонять у дзвіночок, коли ходять. Припустимо, ти реалізував у класі все до дрібниць.

🎮
Уяви: у тебе є повноцінний клас Cow з усіма методами — ходити, їсти, мукати і навіть дзвонити у дзвіночок!

А тут приходить замовник проєкту і каже, що хоче випустити новий рівень гри, де всі дії відбуваються в морі, а головним героєм буде кит.

Ти почав проєктувати клас «Кит» і зрозумів, що він лише трохи відрізняється від класу «Корова». Логіка роботи обох класів дуже схожа, і ти вирішив використати успадкування.

Клас «Корова» ідеально підходить на роль класу-батька, там є всі необхідні змінні та методи. Досить лише додати киту можливість плавати. Але є проблема: у твого кита є ноги, роги і дзвіночок. Адже ця функціональність реалізована всередині класу «Корова». Що тут можна зробити?

💡 Рятівник
До нас на допомогу приходить перевизначення (заміна) методів. Якщо ми успадкували метод, який робить не зовсім те, що потрібно нам у нашому новому класі, ми можемо замінити цей метод на інший.

Як же це робиться? У нашому класі-нащадку ми оголошуємо такий самий метод, як і метод класу-батька, який хочемо змінити. Пишемо в ньому новий код. І все — ніби старого методу в класі-батьку й не було.

Ось як це працює:

C# — Визначення класів 
class Cow
{
    public virtual void PrintColor()
    {
        Console.WriteLine("Я — біла");
    }

    public virtual void PrintName()
    {
        Console.WriteLine("Я — корова");
    }
}

class Whale : Cow
{
    public override void PrintName()
    {
        Console.WriteLine("Я — кит");
    }
}
Тут визначено два класи: Cow та Whale.

Whale успадкований від Cow.

У класі Whale перевизначено метод PrintName() за допомогою ключового слова override.
🔷 Важливо для C#
У C# метод батьківського класу, що буде перевизначено, потрібно позначити ключовим словом virtual, а перевизначений метод у похідному класі — словом override. Без virtual метод не можна перевизначити!
C# — Виклик методу корови 
static void Main(string[] args)
{
    Cow cow = new Cow();
    cow.PrintName();
}
📟 Виведе на екран:
Я — корова
C# — Виклик методу кита 
static void Main(string[] args)
{
    Whale whale = new Whale();
    whale.PrintName();
}
📟 Виведе на екран:
Я — кит

Після успадкування класу Cow та перевизначення методу PrintName, клас Whale фактично містить такі дані та методи:

C# — Ефективний вигляд Whale 
class Whale
{
    public void PrintColor()
    {
        Console.WriteLine("Я — біла");
    }

    public void PrintName()
    {
        Console.WriteLine("Я — кит");
    }
}
Про жодний старий метод ми й не знаємо. 😊

Ось так виглядає клас Whale після застосування успадкування та перевизначення методу. Старого PrintName з «Корова» немає й сліду!

Приведення типів 🔀

👨‍🏫
Викладач
— Перш ніж іти далі, давай згадаємо, як працює перетворення базових типів. Пам'ятаєш, як ми клали int у double і навпаки?
🤖
Аміго
— Здається, так. Там щось про втрату даних і дужки...

Давай освіжимо це в пам'яті. У програмуванні існують поняття розширення (Upcasting) та звуження (Downcasting) типів. Вони потрібні для того, щоб ми могли гнучко і, головне, безпечно працювати з пам'яттю комп'ютера.

Примітивні типи Приклад та пояснення
Upcasting (Розширення)
Безпечне перетворення 
int a = 5;
double b = a; // Неявне перетворення
Маленька "коробка" (int) легко і гарантовано поміщається у велику (double). Дані не втрачаються (число 5 безпечно стає 5.0). Тому компілятор робить це сам, неявно.
Downcasting (Звуження)
Потребує явного підтвердження 
double x = 5.99;
int y = (int)x; // Явне перетворення!
Велика "коробка" не влазить у маленьку. Є ризик втрати даних (дробова частина .99 відкинеться, і y стане просто 5). Тому компілятор змушує нас написати (int), щоб ми взяли відповідальність за можливу втрату даних на себе.
✨
А тепер головна новина: з об'єктами (класами) все працює абсолютно за тим самим принципом!

Розширення та звуження типів 📐

Для посилальних типів, тобто класів, приведення типів працює не так, як для примітивних типів. Хоча у посилальних типів теж є розширення та звуження типу. Приклад:

Розширення типу Опис 
Cow cow = new Whale();
 Класичне розширення типу. Тепер кита узагальнили (розширили) до корови, але у об'єкта типу Whale можна викликати лише методи, описані в класі Cow.

Компілятор дозволить викликати у змінної cow лише ті методи, що є у її типу — класу Cow.
Звуження типу Опис 
Cow cow = new Whale();
if (cow is Whale whale)
{
    whale.PrintName();
}
 Класичне звуження типу з перевіркою. Змінна cow типу Cow зберігає посилання на об'єкт класу Whale.

Ми перевіряємо, що це саме так, і виконуємо операцію перетворення (звуження) типу. Або як її ще називають — downcast.

У C# сучасний спосіб — оператор is з одночасним присвоєнням. 
Cow cow = new Cow();
Whale whale = (Whale) cow; // виняток!
 Посилальне звуження типу можна провести і без перевірки типу об'єкта.

При цьому, якщо у змінній cow зберігався об'єкт не класу Whale, буде згенеровано виняток — InvalidCastException.
✨ Порада для C#
У сучасному C# найбезпечніший спосіб звуження типу — оператор as:

Whale whale = cow as Whale;

Якщо приведення неможливе — whale буде null, а не виняток. Завжди перевіряй на null після as!

Виклик оригінального методу 🎁

Іноді тобі хочеться не замінити успадкований метод на свій при перевизначенні методу, а лише трохи доповнити його.

У цьому випадку дуже хочеться виконати в новому методі свій код і викликати цей самий метод, але базового класу. І така можливість у C# є. Робиться це так: base.Method().

Приклади:

C# — base.PrintName() 
class Cow
{
    public virtual void PrintAll()
    {
        PrintColor();
        PrintName();
    }

    public virtual void PrintColor()
    {
        Console.WriteLine("Я — білий");
    }

    public virtual void PrintName()
    {
        Console.WriteLine("Я — корова");
    }
}

class Whale : Cow
{
    public override void PrintName()
    {
        Console.Write("Це неправда: ");
        base.PrintName();

        Console.WriteLine("Я — кит");
    }
}
C# — Виклик 
static void Main(string[] args)
{
    Whale whale = new Whale();
    whale.PrintAll();
}

Я — білий
Це неправда: Я — корова
Я — кит
У C# для виклику методу батьківського класу використовується base.Метод().
Аналогічно, як і викликаються конструктори батьківських класів!
